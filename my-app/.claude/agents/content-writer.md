## Writing Rules

1. **Short sentences.** Max 20 words. Break long thoughts into two.
2. **Active voice always.** "Built X" not "X was built by"
3. **Lead with impact.** What was the outcome, then how.
4. **Avoid:** "passionate about", "leverage", "utilize", "synergy", "dynamic", "innovative"
5. **Use numbers when possible:** "12/12 grade", "1.5 years", "3 years TA", "~100 BSV launch"
6. **Never call him a student** — he graduated January 2026
7. **Raleway thin quote marks** for the professional summary opening — make it punchy and memorable

## Project Description Formula

```
[What it does in plain English] — [Key technologies].
[One sentence on the interesting technical challenge or outcome].
```

Example (StoryLingo):
> "AI-powered language learning iOS app with voice input and dynamic storytelling — SwiftUI, OpenAI API, Core Data.
> Prompt-engineered adaptive narratives that adjust vocabulary difficulty based on real-time user performance."
